# Changelog

本文档记录 PairNut 的用户可见变更。

## v1.1.2

### 新增

- 新增 OpenCV 图片特征匹配能力。
  - 导入六面图后会提取颜色、纹理和形状特征，并保存到本地数据库。
  - “候选配对”会在双方有图片特征时显示图片相似度，作为边、肚、高、重量之外的辅助证据。
- 新增模型管理能力。
  - 菜单新增“模型管理”和“打开模型目录”。
  - 内置 `OpenCV 基础特征`，无需下载模型即可使用图片匹配。
  - 预留可选 AI 图片模型清单，支持展示模型大小、资源消耗和预期效果。
  - 支持后续可选模型下载、删除和启用，为 AI 图片特征模型接入做准备。
- 新增核桃图片删除功能。
  - 用户可在图片管理入口删除单张已导入图片。
  - 删除图片时同步清理对应图片记录和特征值，避免本地文件缺失后再次打开软件出错。

### 改进

- 明确 PairNut 的通用配对架构原则。
  - 不强制绑定某一种输入方式，用户可以只录入边、肚、高和重量，也可以叠加图片特征，后续还可叠加 3D 模型证据。
  - 配对评分会根据已有证据动态组合，未提供图片或模型时仍可继续使用基础尺寸配对。
- 图片特征依赖改为直接要求 OpenCV 可用，避免静默降级导致用户误以为图片匹配已经生效。
- 更新 README，补充图片匹配、模型目录、模型管理和后续 3D 模型方向说明。
- 新增 PairNut 发版流程 skill，固化版本号、CHANGELOG、tag、GitHub Release 和 Gitee Release 的发布步骤。

### 注意事项

- 当前可直接使用的是 OpenCV 基础图片匹配；可选 AI 模型还没有发布真实模型文件和下载地址。
- 切换到未来的可选模型后，需要重新提取已有图片特征，候选配对才会使用新模型的特征值。

## v1.1.1

### 新增

- 支持点击六面图缩略图查看大图预览。
  - “核桃管理”和“候选配对”中的已导入图片都可以点击放大。
  - 预览窗口会按窗口尺寸等比缩放图片，方便检查细节。

### 改进

- 改进 GitHub Release 发布说明生成流程。
  - Release 正文会优先使用 `CHANGELOG.md` 中对应版本的产品说明。
  - 同时追加 GitHub 自动生成的 `What's Changed`、贡献者和完整变更链接，保留外部贡献者署名。
- 打包产物文件名增加不带 `v` 的版本号，例如 `PairNut-1.1.1-windows-x64.exe`。
- 修复 GitHub Release job 在没有 `.git` 目录时创建 Release 失败的问题。

## v1.1.0

### 新增

- 新增六面图批量导入功能。
  - 支持按 `核桃编号-序号` 命名批量导入，例如 `NJS-01-1.JPG`、`NJS-01-2.JPG`。
  - 序号支持 `1` 到 `6`。
  - 支持 `.jpg`、`.jpeg`、`.png`、`.webp`、`.bmp`、`.heic` 图片格式。
  - 导入时自动匹配当前品种下已有的核桃编号，无法匹配或命名不规范的图片会给出提示。
- 新增六面图缩略图展示。
  - “核桃管理”列表中可直接查看每颗核桃的六面图导入情况。
  - “候选配对”中可同时查看当前核桃和候选核桃的图片，辅助人工判断。
  - 缺失图片位置显示序号占位，方便补拍和检查缺图。
- 新增用户数据目录管理。
  - 数据库和图片统一保存在用户数据目录，不放在软件安装目录中。
  - 菜单新增“打开数据目录”，方便备份和管理图片。
- 新增更新检查。
  - 应用可检查 Gitee Releases 上的新版本。
  - 有新版本时提示用户下载更新。

### 改进

- 打包后的默认数据目录改为按系统区分：
  - macOS: `~/Library/Application Support/PairNut/`
  - Windows: `C:\ProgramData\PairNut\`
  - Linux: `$XDG_DATA_HOME/PairNut/` 或 `~/.local/share/PairNut/`
- 开发环境继续默认使用项目内 `data/` 目录，方便本地调试。
- 移除从开发目录自动迁移数据到打包目录的逻辑，避免误把开发数据带到正式用户环境。
- 默认以最大化窗口显示应用，提升桌面端使用体验。
- GitHub Actions 打包产物文件名增加版本号，例如 `PairNut-1.1.0-windows-x64.exe`。
- GitHub Release 创建流程显式指定仓库，避免 release job 没有 `.git` 目录时报错。

### 修复

- 修复锁定配对历史记录的数据库唯一索引问题，允许同一对核桃多次“锁定 -> 解锁 -> 再锁定”。
- 更新 CI 测试环境配置，避免测试误碰真实本地数据目录。
- 更新依赖和 GitHub Actions 版本，提升构建稳定性。

## v1.0.0

### 新增

- 首个桌面版本发布。
- 基于 PySide6 提供本地桌面界面。
- 使用 SQLite 作为本地数据库。
- 支持品种管理：
  - 创建、编辑、删除品种。
  - 为每个品种配置编号前缀和统一尺寸偏差。
- 支持核桃管理：
  - 创建、编辑、删除核桃。
  - 记录编号、边、肚、高、克重、瑕疵和备注。
  - 支持自动编号和手动编号。
- 支持候选配对：
  - 按同品种内核桃生成候选配对。
  - 基于边、肚、高偏差、克重接近度和瑕疵扣分计算推荐分。
  - 每颗核桃展示高分候选，辅助商家人工判断。
- 支持配对操作：
  - 锁定满意配对。
  - 解除锁定。
  - 拉黑不合适组合，后续推荐中自动排除。
- 新增桌面应用打包流程，支持生成 Windows 和 macOS 安装包。
- 新增 GitHub Actions CI、打包、安全扫描、依赖审计和 issue/PR 模板。

### 改进

- 从早期 Flet 界面迁移到 PySide6，界面更适合本地桌面工作流。
- 移除示例数据库文件的版本控制，避免提交本地业务数据。
- 增加应用图标和基础品牌视觉。
